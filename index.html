🏗️ Componentes Core

  • Backend: FastAPI (Python) - API REST asíncrona
  • Frontend: React + Vite - SPA con hot reload
  • IA Engine: Ollama server + Llama 3.2 local
  • Database: Firebase Firestore + real-time sync
  • Auth: Firebase Auth + JWT tokens

🔄 Data Flow

  • React UI → FastAPI endpoints (REST)
  • FastAPI → Ollama server (HTTP)
  • Ollama → Llama 3.2 (local inference)
  • Response → Firebase (persistence)
  • Dashboard → Real-time updates

📋 Transferencia: Implementación

⚙️ Requisitos del Sistema

  • OS: Ubuntu 20.04+ / CentOS 8+ / Windows Server
  • Python: 3.8+ con pip y virtualenv
  • Node.js: 18+ con npm/yarn
  • Hardware: 16GB RAM, GPU recomendada (CUDA/ROCm)
  • Storage: 50GB para modelos y datos

🔧 Setup Técnico

  • Ollama install: curl + modelo Llama 3.2 download
  • Backend: uvicorn --host 0.0.0.0 --port 8000
  • Frontend: npm build → nginx/apache deploy
  • Firebase: Service account + Firestore setup
  • SSL/TLS: Certificados para producción

🚀 Timeline de Deployment

  • Día 1: Infraestructura + Ollama + modelo download
  • Día 2: App deployment + Firebase config
  • Día 3: Testing completo + training del equipo

🛠️ Troubleshooting Común

  • Ollama no responde: Check GPU memory, restart service
  • Context loops: Review XOR logic, adjust prompts
  • Firebase timeouts: Verify connection, check DB indexes
  • High latency: GPU optimization, model quantization
